## Suspected defects found during the code study (verify first-class)

1. **`annualBudgetPeriods()` and `weeklyRatioFromReadiness()` (Allocate.jsx) still read
   `grids[]`** — but since the calendar purge, grids ship all `-1` from FirstRun and
   TeachingProfile. So weekly period count derives to 0: budget methods "weeks"/"days" likely
   compute 0/None, and the G4 ratio always falls back to the default period rows (counts 45/60 —
   themselves odd defaults). Neither function reads the new `periods_per_week`/`ppw_by_duration`.
2. **Allocation localStorage cache `allocations_{subject}_{grade}` is not user-scoped** — and the
   server register only *overrides* it when non-empty. Same-browser user switch may paint user
   A's allocations for user B.
3. **FirstRun preview dead-end** — when a subject·grade has *no* saved plans at all,
   `previewError` shows, the CTA is disabled, and there is no Back on the preview step (one-way
   by design). Teacher appears stuck without a refresh.
4. **Portal exit destination** — CLAUDE.md §0 says "a portal visit always exits to My Classes,
   never the profile accordion", but TeachingProfile's manage flows end with `setScreen("view")`
   (the accordion). Verify live; flag whichever is true as spec-vs-code divergence.
5. **`/plans/{subject}/{grade}/{filename}/view` has no filename traversal guard** (unlike
   `_plan_key` used by archive/prepared). Probe with `..%2F` style filenames.
6. Known, pre-existing (re-verify only, don't re-discover): Science IX ch_02 renders LU1 with
   "No phases recorded" + "STAGE NONE" (plan-data issue, MEMORY 2026-07-06 round-5).

- Full 12-step tour: step transitions perform real moves (2→3 My Lessons, 4→5 back, 6→7 REAL
  attach, 8–9 lesson view, 10–11 demo-complete render-only — real pointer untouched, 11→12
  profile + gear ring). Back at every boundary reverses cleanly (7→6 unbinds; 12→11 popup

  recently PREPARED lesson, never a raw library entry.
- Teach loop: attach via "+" (picker lists prepared-only, excludes bound); card shows LU rail;
  mark unit complete → Undo (session-only) → next-up card; pointer survives refresh AND
  sign-out/sign-in (server section-state reconcile); last unit → "Mark chapter complete" → done

  "completed" history row and opens picker; untrack ("−") with 0 units → NO history row
  (anti-noise gate); untrack with ≥1 unit → "untracked" row with progress; history popup overlays
  the live chapter as Ongoing only when ≥1 unit done.
- Cross-tab agreement: My Lessons card colours + status lines ("Teaching now 6A", "Completed…")
  match My Classes state; 20s/focus sync doesn't clobber an open modal.
