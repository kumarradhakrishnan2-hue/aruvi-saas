Aruvi · Chapter Organization · Social Sciences (edge model)

Competency is not a spine — it is a weave

Under the rewritten middle-SS constitutions, a unit carries zero, one, or several competency edges, and each mapped competency reaches many units. Grouping units under competency accordions (the current SS org page) becomes dishonest: a unit with three edges would live in three folders, and a unit with none would live nowhere. These concepts keep the plan's teaching order as the one true spine and treat competency as an overlay. All four are drawn from the real edge-model plan — Grade VII, Ch. 4, New Beginnings — Cities and States (11 units, pointer on unit 5).

Concept 1 · quietest

Teaching rail + skill tags

The accordion goes away for SS. Units sit flat in teaching order (the pointer's home); each card carries small competency tags coloured by weight tier — Central filled, others outlined. Tapping a tag in the legend lights that competency's units: the one-to-many view on demand, zero new geometry.

Concept 2 · signature

The loom — threads in the margin

The marginal rail is Aruvi's signature; here it becomes a loom. Each competency is a vertical thread beside the unit list; a dot marks a unit that genuinely builds it. Threads enter at their first unit and leave at their last, so the chapter's competency arc is visible as shape — C-2.1 runs nearly the whole chapter, C-5.1 touches one unit. Tap a thread head to isolate it.

Concept 3 · two lenses

In class / What it builds

A lens toggle instead of one forced axis. In class is the flat teaching rail. What it builds inverts the graph: the chapter's final competency set as a ledger — each competency with its weight tier, a mini unit-rail showing exactly which units realise it, and its edges (LO + cognitive demand) on open. This is also where a competency_gap_note would live.

Concept 4 · bipartite flow

Units ↔ Competencies, both ends visible

The Sankey idea, adapted for the edge model and a portrait phone. Ribbons are connections, not quantities — a unit's 40 minutes is never split across its edges (weights are emphasis, never arithmetic), so ribbon width follows tier, and competency cards count units, not minutes. Weight tier is the plain name with the allocation report's dots beneath (●●● Central · ●● Substantive · ● Present) — never a colour. All ribbons rest faint; tap either end and only its ribbons draw, with a popup below the tapped card: the competency's full text, or the unit's number, name and time. Zero-edge units show a quiet “—”.

Data-handling notes (all concepts)

Zero-edge units are first-class. Units 5, 10 and 11 carry no edges — they are taught (consolidation, review, present-day connection) but generate no LO. They render as plain cards: no tag ≠ not taught. Never invent a "General" bucket for them.

Teaching order stays the contract. The flat rail is the same flattenUnits walk the pointer uses (test_unit_order.py); competency is always an overlay, never a regrouping.

Weight tiers, never weight integers — and never colour. Central / Substantive / Present on screen as plain names with the allocation report's dot convention beneath (●●● / ●● / ●). Colour stays reserved for competency identity (threads, ribbons, codes), so tier and identity never compete for the same channel.

Section anchors become the card's context line. The v2.7 plans carry real section_anchors, but at ~1 section per unit an accordion of sections is chrome without payoff — shown here as the small mono line under each title instead.

Thread colours. C-2.1 pine, C-3.1 clay, C-3.2 ochre, plus two proposed muted tokens (slate, plum) for the 4th/5th competency. SS mappings run ~5 primaries, so five hues suffice; beyond that, tier colouring (Concept 1) degrades gracefully.

Old-model plans (single competency per period, e.g. viii ch_04) keep the current contiguous-run competency accordion — the port branches on which key the period carries.

Why not a true Sankey. Sankey grammar promises conservation of flow — a quantity partitioned downstream. Edges aren't quantities: a 3-edge unit does not split its 40 minutes, and per-competency minute totals would double-count (this chapter would show ~3× its real time). Concept 4 keeps the bipartite geometry but demotes ribbons to connections (width = tier), counts units instead of minutes, and uses tap-to-focus so 14 crossing ribbons stay legible at 360 px.

Combination worth considering: Concept 3's toggle with Concept 2's loom as the "In class" lens — the weave while teaching, the ledger when reflecting.
