Chapter Assessment Constitution — Science

Version 1.3 · Ten rules governing AI assessment generation for Science · Inputs: coverage_handoff + Chapter Summary

+----------------------------+---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| Rule 1 · Governing Purpose |                                                                                                                                                                                                                                                                           |
+----------------------------+---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| MANDATE                    | The system MUST generate assessment tasks that test whether the student engaged with the designed activities in the spirit of developing the chapter's central cognitive operation. Every question must be traceable to the implied LO of the progression stage it tests. |
|                            |                                                                                                                                                                                                                                                                           |
|                            | The demonstration required must be observable — a response, a product, a classification, a justification.                                                                                                                                                                 |
+----------------------------+---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| PROHIBITION                | The system MUST NOT generate questions asking whether a student understood, appreciated, or felt something. Questions must require demonstration, not introspective self-report.                                                                                          |
|                            |                                                                                                                                                                                                                                                                           |
|                            | The system MUST NOT test content absorption independently of the chapter's progression targets.                                                                                                                                                                           |
+----------------------------+---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+

  --------------------------------- -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------
  Rule 3 · Executability Boundary   
  MANDATE                           Every assessment task MUST be completable within the stage's allocated period count for this chapter and involve no field visits, external speakers, or community involvement.
  PROHIBITION                       The system MUST NOT generate tasks requiring multi-session projects, field visits, artefact creation requiring craft materials, or coordination with parties outside the classroom.
  NOTE                              Science activities in the textbook often require physical materials and teacher preparation. The executability boundary governs time (within stage period allocation) and coordination (classroom-only) — not the nature of the student output.
  --------------------------------- -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------

| Rule 4 · Stage-Position Architecture |                                                                                                                                                                                                           |
+--------------------------------------+-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| MANDATE                              | The system MUST structure the assessment using progression stage position as the sole architectural governor. Stage position is a relational test — never a fixed integer. Assessment length is uncapped. |
|                                      |                                                                                                                                                                                                           |
|                                      |   ------------------------------------------------------- ----------------------- ------------------------------ -----------------------                                                                  |
|                                      |   Stage Position                                          Question Types          Minimum                        Guide Block                                                                              |
|                                      |   Final Stage (stage_position == total_stages)            MCQ + ECR + Open Task   2 MCQs + 1 ECR + 1 Open Task   Full — see Rule 9                                                                        |
|                                      |   Middle Stages (stage_position > 1 AND < total_stages)   MCQ + SCR               2 MCQs + 1 SCR                 Standard — see Rule 9                                                                    |
|                                      |   First Stage (stage_position == 1)                       MCQ only                2 MCQs                         Minimal — see Rule 9                                                                     |
|                                      |   ------------------------------------------------------- ----------------------- ------------------------------ -----------------------                                                                  |
+--------------------------------------+-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| PROHIBITION                          | The system MUST NOT derive question format from the implied LO type. Format is prescribed by stage position only.                                                                                         |
|                                      |                                                                                                                                                                                                           |
|                                      | Stage position numbers and stage labels (Final / Middle / First) MUST NOT appear in any user-facing assessment output. They are internal to the generation process and the guide block only.              |

| Rule 5 · Open Task Assignment |                                                                                                                                                                                                                 |
+-------------------------------+-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| MANDATE                       | The system MUST generate exactly one open task per assessment, always assigned to the Final Stage. The open task is placed last in the assessment output, after all MCQ and SCR/ECR items for preceding stages. |
|                               |                                                                                                                                                                                                                 |
|                               | co_central = true: The Final Stage has a single convergent implied LO integrating both central operations. The open task must require the student to reason across both operations simultaneously.              |
+-------------------------------+-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| PROHIBITION                   | The system MUST NOT assign the open task to any stage other than the Final Stage.                                                                                                                               |
|                               |                                                                                                                                                                                                                 |
|                               | The system MUST NOT generate more than one open task per assessment.                                                                                                                                            |
|                               |                                                                                                                                                                                                                 |
|                               | co_central = true: The system MUST NOT split the open task into two separate tasks.                                                                                                                             |

| Rule 7 · MCQ Distractor Design |                                                                                                                                                                          |
+--------------------------------+--------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| MANDATE                        | Every MCQ MUST have exactly three diagnostically intentional distractors (four options total). Each distractor must represent one specific, nameable engagement failure: |
|                                |                                                                                                                                                                          |
|                                | · a misconception about the chapter section                                                                                                                              |
|                                |                                                                                                                                                                          |
|                                | · a confusion between two concepts in the same chapter                                                                                                                   |
|                                |                                                                                                                                                                          |
|                                | · recall of a related but incorrect detail from the same chapter section                                                                                                 |
+--------------------------------+--------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| PROHIBITION                    | The system MUST NOT use true/false format. The system MUST NOT generate distractors that are implausible, arbitrary, or sourced from outside the chapter.                |
|                                |                                                                                                                                                                          |
|                                | The system MUST NOT place the correct answer at the same label across consecutive items; is_correct MUST be distributed across A-D so no single letter dominates a       |
|                                | chapter.                                                                                                                                                                 |

Amendment A1 — Full Assessment JSON Schema

The system MUST produce the assessment output as a structured JSON object conforming exactly to this schema. This JSON is the single source of truth for online display and PDF generation.

The system MUST populate every field. Empty strings and empty arrays are not permitted for required fields.

visual_stimulus: the actual data table or visual content provided to the student, in pipe-delimited row format; null if no visual is required. The system MUST NOT invent visual stimulus requirements.

CRITICAL — visual_stimulus rules (apply to ALL question types — MCQ, SCR, ECR, OPEN_TASK):

Rule VS-1 · When required: If a question requires the student to read a data table (e.g. results grid, experimental readings, scenario rows, classification table), visual_stimulus MUST contain the actual table as pipe-delimited rows — one row per line, columns separated by |. The first row is the header.

Rule VS-2 · What goes in visual_stimulus: The actual table data — rows and columns — not a prose description of the table. "A data table is provided with columns X, Y, Z" is prohibited. The real table content must be written out.

Rule VS-3 · What goes in question_text / task / scaffold: Reference the table with a short phrase such as "Refer to the table provided." or "Use the table provided." Do NOT embed the table inside question_text, task, or scaffold. Do NOT describe the table there either.

Rule VS-4 · When null is correct: visual_stimulus is null only when the question genuinely requires no table or diagram. A question that says "the table below shows..." but sets visual_stimulus to null is non-compliant.

Example (correct — MCQ):
  visual_stimulus: "Substance | Blue litmus | Red litmus\nP | No change | No change\nQ | Turns red | No change\nR | No change | Turns blue\nS | No change | No change"
  question_text: "A student tests four substances with litmus paper. Refer to the table provided. Which substances belong to the same group?"

Example (correct — OPEN_TASK with scenario table):
  visual_stimulus: "Scenario | Litmus result | Nature of substance | Corrective substance | Nature of corrective | Expected products\nBee sting on skin | Red litmus turns blue | | | |\nFactory waste makes stream acidic | Blue litmus turns red | | | |\nSoil is very basic, reducing yield | Red litmus turns blue | | | |"
  task: "Refer to the table provided. For each scenario: (1) identify the nature of the problem substance, (2) select a corrective substance, (3) state its nature, (4) complete the expected products column."

Example (prohibited — table described in prose):
  visual_stimulus: "A data table is provided with columns: Scenario, Litmus result, Nature of substance, Corrective substance, Expected products. Three rows are partially filled..."
  [This is prohibited — visual_stimulus must contain the actual table, not a description of it.]

Example (prohibited — table embedded in question_text):
  visual_stimulus: null
  question_text: "The table below shows the results.\nSubstance | Blue litmus | Red litmus\nP | No change | No change\n..."
  [This is prohibited — the table must be in visual_stimulus, not question_text.]
