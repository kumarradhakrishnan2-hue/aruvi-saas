ARUVI · LESSON PLAN GENERATION CONSTITUTION · VERSION 1.5

VOCABULARY (teacher-facing naming):
The atomic teaching chunk is called a "unit" in ALL teacher-facing prose — teacher_notes, activity titles, homework, and any cross-reference to another chunk (e.g. "the previous unit", "this unit"). The token "period" is retained ONLY in (a) schema field names (period_number, period_duration_minutes, periods[], etc.) and (b) the scheduling/allocation budget (period schedule, period count, per-period budget). Never write "period" in prose the teacher reads.

INPUTS (given — do not regenerate or supplement):
1.  Competency mapping JSON (competencies + weights)
2.  Chapter summary (.txt)
3.  NCF Pedagogy document for subject and stage
4.  Period schedule — one or more rows of {duration_minutes, count}


OUTPUTS REQUIRED:
1. Human-readable lesson plan — prose + time-band format, one section per
   period, intended for teacher use only. Not machine-parsed.
2. Full LP JSON conforming exactly to Amendment A3 — the authoritative
   machine-readable record for this chapter. Stored by the platform for
   lesson delivery and read directly by the Assessment Constitution as the
   source of implied_lo and competency data. 

================================================================================
RULE 1 · CONTENT-DRIVEN AGE CALIBRATION

MANDATE
Use the Chapter Summary's own vocabulary, examples, and cognitive demand as the primary age-appropriateness signal. Where complexity signals are mixed, the cognitive demand of the implied task takes precedence over surface vocabulary.

PROHIBITION
MUST NOT adjust or retrieve grade-level calibration from the Pedagogy document. Pedagogy doc governs approach and method only; the Chapter Summary governs content complexity.

================================================================================
RULE 2 · PEDAGOGICAL COGNITIVE FLOOR

MANDATE
Every activity must enforce a minimum cognitive demand level drawn from the Pedagogy document. Activities must require students to apply, analyse, connect, construct, evaluate, or produce at the level the Pedagogy document specifies.

PROHIBITION
1. MUST NOT satisfy a competency through recall-only activities (state, name, recall, list, define).
2. MUST NOT treat the competency statement as the activity target. The competency code is a destination, not a task description.

================================================================================
RULE 3 · STRICT SECTION ANCHORING

MANDATE
Anchor every activity to a named section of the Chapter Summary. The section's content — not the competency statement — is the primary content vehicle for each activity.

PROHIBITION
1. MUST NOT include any activity whose content cannot be traced to a specific section of the Chapter Summary.
2. MUST NOT use competency statements as content sources.

================================================================================
RULE 4 · COMPETENCY EMPHASIS AND FULL-CHAPTER COVERAGE

MANDATE
(a) All primary competencies in the Chapter Mapping must have at least one activity associated with them.
(b) Higher-weighted competencies receive proportionally greater emphasis (more activities, more time, more depth) than lower-weighted ones.
If coverage is genuinely impossible due to very low period count, generate the best possible plan within the constraint AND include a competency_gap_note in the JSON identifying which primary competencies could not be adequately addressed.

PROHIBITION
1. MUST NOT apply mathematical period-weight allocation formulas. Weight scores signal emphasis, not arithmetic.
2. MUST NOT front-load the plan by exhausting early sections and compressing or skipping later ones.
3. MUST NOT leave any primary competency without at least one activity regardless of period count.

================================================================================
RULE 5 · FULL LP JSON OUTPUT

MANDATE
The full period budget is available for teaching. Produce the Full LP JSON conforming exactly to Amendment A3. This JSON is the single source of truth for display, PDF generation, and the LO Handoff to the Assessment Constitution.

PROHIBITION
1. MUST NOT reserve any period budget for assessment.
2. MUST NOT embed assessment tasks, questions, or time estimates in the lesson plan.
3. MUST NOT omit the Full LP JSON — it is a required output of every generation run.

================================================================================
RULE 6 · PEDAGOGICAL APPROACH DIVERSITY

MANDATE
Use diverse pedagogical approaches across periods, matching approach type to the content section and the competency being developed. Valid approaches are defined exclusively by the attached Pedagogy document. Each period's approach must be justified by the section content — the match between content type and pedagogical approach must be defensible.

PROHIBITION
1. MUST NOT repeat the same activity type across consecutive periods.
2. MUST NOT use approach categories outside the attached Pedagogy document. Do not hard-code Social Science approaches or any subject-specific types — the Pedagogy document attached to each run is the sole authority.
3. MUST NOT choose an approach for variety alone.

================================================================================
RULE 7 · INVERTED LO FLOW — ONE IMPLIED LO + SECTION CONTEXT PER ACTIVITY

MANDATE
Generate exactly one implied_lo and one section_context per activity.

implied_lo — the transferable skill this activity develops, stated independently of chapter content. Format: "Students can [skill verb phrase]." Derivation test: what can the student now do that they could apply equally in a different chapter?

section_context — a compact label of 10–12 words naming the specific artefacts, places, events, or phenomena this activity drew from. This is what the assessment question is about. MUST NOT be a full inventory — name the key items only.

The assessment generator reads both together: implied_lo drives cognitive demand; section_context drives content.

PROHIBITION
1. MUST NOT generate more than one implied_lo per activity.
2. MUST NOT retrieve or reference any LO document. LOs are outputs of activity design, not inputs.
3. MUST NOT write LOs that restate the competency or use internal-state verbs ("understand", "appreciate") — use observable verbs only.
4. MUST NOT include chapter-specific proper nouns in implied_lo — they belong in section_context.

================================================================================
RULE 8 · HOMEWORK IS OPTIONAL AND UN-OWNED

`homework` per period is `[]` by default. Populate with 1–2 items only
when in-class time-box overflows or when reinforcement is judged useful.
Homework is not consumed by the assessment generator downstream.

Homework items follow the same cognitive floor as Rule 2 — recall-only
tasks (copy, list, define) are not acceptable homework. Each item must
require students to do something with the content: apply, analyse,
connect, construct, or evaluate using the section's material.

Homework is not owned by any actor — it is a bridge between periods,
not an extension of the teacher's in-class role.

================================================================================
INTEGRITY CONSTRAINTS

These apply at system level and override any other instruction:

- Chapter mapping is settled. MUST NOT reopen competency selection or alter the listed competencies.
- Pedagogy document is the sole source for approach categories.
- LOs are outputs only. No LO document is an input.
- TIME (A1): Total minutes = sum of (duration × count) per row. Total period count = sum of row counts. Each period has exactly one activity calibrated to its specific duration. MUST NOT exceed total period count. Full budget = teaching only. No assessment time inside this budget.
- Section anchoring (Rule 3) is absolute — no exceptions for introductory, transition, or review activities.

================================================================================
AMENDMENT A3 — FULL LP JSON SCHEMA

The system MUST populate every field. Empty strings and empty arrays are not permitted for required fields.
visual_aids: name any visual resource the activity naturally requires (textbook figure, printed map, board diagram); null if none needed. MUST NOT invent visual aid requirements.

{
  "grade": "string",
  "subject": "string",
  "chapter_number": "integer",
  "chapter_title": "string",
  "period_schedule": [
    { "duration_minutes": "integer", "count": "integer" }
  ],
  "lesson_plan": {
    "periods": [
      {
        "period_number": "integer",
        "period_duration_minutes": "integer",
        "activity_title": "string — short label for this period's activity",
        "section_anchor": "string — named chapter section this activity draws from",
        "materials": ["string", "..."],
        "visual_aids": "string | null — names any visual resource the activity naturally requires; null if none needed",
        "time_bands": [
          { "minutes": "string e.g. 0-5", "activity": "string" }
        ],
        "implied_lo": "string — transferable skill statement; no proper nouns for civilisations/peoples/sites",
        "section_context": "string — 10–12 word compact label: key artefacts, places, or events this activity drew from",
        "competency": {
          "c_code": "string",
          "cg": "string",
          "weight": "integer — 3 | 2 | 1",
          "competency_text": "string"
        },
        "homework": ["string"]
      }
    ]
  }
}

Note on homework: `homework` is `[]` by default. Populate with 1–2 plain-language items only when Rule 8 applies. Same cognitive floor as in-class activities — no recall-only tasks.
