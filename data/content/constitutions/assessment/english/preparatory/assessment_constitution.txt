RULE 1 | PRESENT SPINES ONLY
================================================================================

Emit only spines that have at least one `section_contribution` in
the LP handoff. A spine with zero contributions is OMITTED entirely
â€” no note, no placeholder.

Present spines are emitted in this fixed order (skip absent ones):
  reading Â· oracy Â· writing Â· word_work Â· beyond_text

Within each spine, items are ordered by `section_id` (A â†’ B â†’ C).


================================================================================
RULE 2 | ONE ITEM PER SPINE-CELL IMPLIED LO
================================================================================

For each `section_contribution` in the handoff, generate exactly ONE
assessment item. `implied_lo` determines question type and cognitive
demand; `section_context` determines what the question is about.
Neither alone is sufficient.

Constraints:
  a) The item MUST be grounded strictly in the section's content
     source (per INPUTS Â§2). No outside content.
  b) The item MUST be original â€” must not reproduce, paraphrase, or
     structurally echo any textbook exercise wording.
  c) The item MUST directly test the `implied_lo` â€” not a generic
     comprehension question, not a tangentially related exercise.
  d) Question type is drawn from the spine's permitted set (Rule 4).
  e) ECR is BANNED at preparatory (Rule 4).

There is no fallback path: a spine cell with no contribution
produces no item. Item count per chapter = total
`section_contributions` across all spines in the handoff.


================================================================================
RULE 3 | RICHNESS â€” GROUND ITEMS IN SECTION CONTENT
================================================================================

Every item must be visibly anchored in the section's actual content.
Generic questions usable across any chapter are prohibited.

Required â€” items must draw on at least ONE of:
  - A named character, speaker, or person from the section.
  - A specific event, scene, or episode from `prose_summary` or
    `picture_story_summary`.
  - A specific line, image, or phrase from `poem_text`.
  - A speech-bubble line from `dialogue_text` (picture_narrative).
  - The vocabulary, phonics, or grammar concept as it appears in the
    section (for word_work items: the concept
    MUST be named explicitly in the stem â€” e.g. "consonant blend
    'gr'", "action words", "compound words").
  - A specific writing prompt or communicative context from the
    section's theme.

Prohibited:
  - Stems answerable without reading the section.
  - Stems recycled from textbook exercise wording.
  - Phonics or vocabulary items that test a concept in isolation
    without grounding in the section's language.


================================================================================
RULE 4 | QUESTION TYPE
================================================================================

Permitted `question_type` values at preparatory:

  MCQ           Multiple Choice (4 options, exactly one correct)
  SCR           Short Constructed Response (1â€“3 sentences)
  MATCH         Matching (Column 1 â†” Column 2)
  FILL_IN       Blanks in a cloze set; at most ONE visual (Rule 9); multi-part
                A/B allowed ONLY if every part is textual (no visual)
  TRUE_FALSE    True / False (with optional justification)
  ORAL_PROMPT   Speaking / listening task with rubric
  WRITING_TASK  Short-form writing task with rubric
  PROJECT       Beyond-text project / hands-on extension

ECR is BANNED at preparatory. Long-form writing is short-form
`WRITING_TASK` (rubric) or `SCR` (1â€“3 sentences). Emitting
`question_type: "ECR"` is a constitution violation.

Default mapping by spine â€” pick the type that best fits the
`implied_lo`:

  reading                       â†’ MCQ Â· SCR Â· TRUE_FALSE Â· FILL_IN Â· MATCH
  oracy                         â†’ ORAL_PROMPT (default) Â·
                                  SCR (short written response to a
                                  listening prompt) Â·

  word_work  â†’ FILL_IN Â· MATCH Â· MCQ Â· TRUE_FALSE Â·

TRUE_FALSE vs MCQ disambiguation:
  TRUE_FALSE â€” every option is a complete declarative sentence the

  MCQ        â€” exactly one option is correct and the options are NOT
               full declarative sentences being judged for truth

Do NOT classify "select the true statements" tasks as MCQ.

RULE 5 | ANSWER LAYER â€” CLOSED VS OPEN
================================================================================

Each item is CLOSED or OPEN by its `question_type`:

  CLOSED  â†’  MCQ Â· FILL_IN Â· MATCH Â· TRUE_FALSE Â· SCR (when factual)
  OPEN    â†’  ORAL_PROMPT Â· WRITING_TASK Â· PROJECT Â· SCR (when reflective)

CLOSED items carry `teacher_guide.suggested_answer` with type-specific
rules:
  MCQ           â€” `suggested_answer` MUST be "" (the correct option is
                  already marked `is_correct: true` in `options[]` and

                  the verification-failure fallback only).

================================================================================
RULE 6 | ANSWER VERIFICATION
================================================================================

For every CLOSED item, verify the answer against the section's content
source:
  - prose / narrative / dialogue / informational â†’ `prose_summary`
  - poem                                         â†’ `poem_text` +
                                                   `poem_appreciation_summary`
  - picture_narrative                            â†’ `picture_story_summary` +

For oracy items that are listening-based (carry a `transcript_ref`):

the section's `oracy` cell (the one allowed read of a tasks_verbatim
sub-field, per INPUTS Â§2 carve-out).

Set `verified: true` only if the answer is unambiguously supported by
the source.

For OPEN items: `verified` is always `true` (rubrics are not solved).

If a closed item's answer cannot be verified, Rule 7 applies.


================================================================================
RULE 7 | FALLBACK (VERIFICATION FAILURE)
================================================================================

When a closed item's answer cannot be verified:

  - `item_stem`                       = ""
  - `verified`                        = false
  - `teacher_guide.suggested_answer`  = ""
  - `teacher_guide.note`              = "[Verification failed]
    Refer to the section text for <spine> content in <section_title>."

The fallback item is still emitted (not silently dropped) so the
teacher sees the gap and can substitute.


================================================================================
RULE 8 | SOURCE TAGGING
================================================================================

Every item carries:

  - `source_section_id`     A | B | C ...
  - `source_section_title`  copied verbatim
  - `source_section_type`   poem | prose | narrative | dialogue |

  - `source_spine`          reading | oracy | writing |
                            word_work | beyond_text
  - `source_lo`             the spine-cell `implied_lo` from the LP
                            handoff, copied verbatim
  - `source_context`        the `section_context` from the LP handoff,
                            copied verbatim

RULE 9 | VISUAL STIMULUS
================================================================================

`visual_stimulus` is a STRING. Permitted formats at preparatory:

  ""                            no stimulus (default for most items)

  Pipe-delimited table          2+ lines, every line contains "|",
                                first row is header. Used for MATCH
                                items and tabular comprehension built
                                from the source text.

For items grounded in a textbook picture (picture_narrative sections,
picture-prompted writing, oral observation prompts), do NOT introduce
a separate visual format. Instead, reference the textbook page in
`item_stem` itself â€” e.g. "Look at the picture on Textbook page 6
and write what each character is doing." The teacher has the book;
the image lives there.

Prohibited: inline SVG, ASCII art, image markup, base64 image,
external image reference, italic prose specs.

MATCH items with a pipe-table `visual_stimulus`: `item_stem` carries
ONLY the task instruction (e.g. "Match each animal with its young
one."). The column contents are entirely in `visual_stimulus`. Do
NOT repeat the column entries in `item_stem`.

FILL_IN items with a pipe-table `visual_stimulus`: the SAME rule
applies. `item_stem` carries ONLY the task instruction; the table â€”
header row and every data/blank row â€” lives entirely in
`visual_stimulus`. Do NOT reproduce the table as pipe-markdown, plain
text, or a paraphrased list in `item_stem`. A FILL_IN item owns at
most ONE `visual_stimulus`. It MAY carry multiple parts (A/B) ONLY if
every part is textual/prose; any part that would need its own table or
visual must be split into a separate item.

MCQ / TRUE_FALSE: the choices live ONLY in `options[]`. `item_stem`
carries only the question or task instruction â€” do NOT inline, letter,
or repeat the option texts (no "A. â€¦ B. â€¦" block) in the stem.

Structural fidelity:
  - Multi-line stems (TRUE_FALSE statement lists only, NOT MCQ
    options) â€” reproduce each line separately, do NOT collapse into
    prose.
  - Tabular structures â€” render as pipe-delimited table in
    `visual_stimulus`.

RULE 10 | TRANSCRIPT REF FOR LISTENING ITEMS
================================================================================

At preparatory, listening lives WITHIN the oracy spine â€” there is no
separate listening spine.

A listening-based oracy item carries `transcript_ref` at the item
level:
  - Format: "p. NN"
  - Lifted verbatim from the relevant task object's `transcript_ref`
    in the section's `oracy` cell.

The transcript text is NOT inlined into the item. The generator
reads `transcript_text` for verification only (Rule 6); the teacher
reads aloud from the textbook at the indicated page.

Non-listening oracy items (free conversation, recitation, picture
discussion, etc.) carry `transcript_ref: ""`.
